## Implementation Guidance

- Use these references to shape the interface rather than rendering the screenshots as decorative content.
- Use the dense dashboard references for app and admin shells: dark navy navigation, white page canvas, compact cards, right-rail summaries, and badge-heavy states.
- Use the discovery references for public search: strong hero/search area, floating filter surface, image-like visual card bands, status chips, and saved controls.
- Use the messaging reference for conversation views: clear participant columns, chat bubbles, timestamps and safe separation between chat and deal state changes.
- Use the escrow/wallet references for deal workspaces: state tables, milestone cards, ledger rows, funding/release badges and security callouts.
- Use the profile/trust reference for public profiles and reviews: verification badges, trust score breakdowns, profile completeness and review cards.
- Do not force duplicate screenshots into the UI. Duplicates are documented and used only to confirm recurring patterns.
- Use `next/image` for actual raster content where it appears in product UI. The current JPEGs are UI references, so they are not rendered as production content images.
